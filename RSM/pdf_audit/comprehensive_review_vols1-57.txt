DEFINITIONS
------------------------------------------------------------------------------

EMPTY
  A .txt file exists but contains no meaningful text (< 10 characters of
  body content after the header line). The extraction created the file but
  failed to capture any article text.
  Action: Re-extract from PDF, or delete if the article is an image/plate.

STUB
  A .txt file exists but is very small (< 500 bytes). Only a fragment of
  the article was captured -- typically just a title, a header, or a few
  words. Some stubs are legitimately short items (CONTENTS pages, BOARD
  listings, frontispiece captions, brief notices).
  Action: Check if the article should have more text. If so, re-extract
  from PDF. If the item is legitimately short, mark as OK.

STUB_POEM
  Same as STUB but the file is categorized as a poem or frontispiece.
  Many poems are genuinely short (a few stanzas = under 500 bytes), so
  not all STUB_POEMs are problems. A prior review confirmed 186 as
  legitimate short poems (STUB_POEM_OK). The remaining need human review
  to determine if they are complete or truncated.
  Action: Read the poem. If it looks complete, ignore. If truncated,
  re-extract from PDF.

WRONG_CONTENT
  A .txt file exists and has substantial text, but the text does not match
  the article title in the header. This is typically boundary bleed --
  the extraction grabbed text from an adjacent article instead of (or in
  addition to) the correct one. Common in Vols 41-50 (wrong content at
  start of files) and Vols 35-40 (multi-article amalgamation).
  Action: Compare file content against title. If wrong, re-extract the
  correct text from the PDF source.

MISSING
  The magazine CONTENTS page lists an article title, but no corresponding
  extracted .txt file could be found in the output directory. The article
  was never extracted from the PDF. No file exists to review -- the content
  must be located in the source PDF and extracted.

  IMPORTANT CAVEAT: The MISSING count has known false positives, especially
  in Vols 41-50. These volumes have extremely truncated file headers (e.g.,
  Title: Individual instead of Individual Liberty and its Rewards), which
  causes the matching algorithm to fail even when the file exists.

  Confidence levels by volume range:
    Vols 1-10:  HIGH   -- good header titles, reliable matching
    Vols 11-20: MEDIUM -- mix of real missing + OCR matching failures
    Vols 21-35: LOWER  -- truncated titles cause some false positives
    Vols 36-40: MEDIUM
    Vols 41-50: LOW    -- very truncated headers, many false positives
    Vols 51-57: HIGH   -- full titles in headers, page-number matching

  Action: Check the source PDF CONTENTS page. If the article exists in
  the PDF but has no extracted file, extract it. If a file does exist
  under a different name, this is a false positive -- ignore.
